    Esta traducción de GPhoto fue revisada el 2021-02-09. Si existen cambios puede actualizarla o avisar al equipo de traducción.

    Libgphoto2 es la principal librería diseñada para permitir el acceso a cámaras digitales por parte de programas externos (frontend), como digiKam y gPhoto2. La lista de cámaras oficialmente soportadas está disponible en su página web (aunque podrían funcionar más).

    Este artículo documenta la configuración de libgphoto2 para acceder a las cámaras digitales. Algunas cámaras se montarán como dispositivos de almacenamiento USB normales y pueden no requerir el uso de libgphoto2.

    Instalación

    Instale el paquete libgphoto2, y opcionalmente gphoto2 para tener una interfaz de línea de comandos.

    Utilización de GPhoto2

    GPhoto2 es un cliente de línea de comandos para libgphoto2. GPhoto2 permite acceder a la librería libgphoto2 desde un terminal o desde un shell script para realizar cualquier operación de cámara que sea posible. Esta es la principal interfaz de usuario.

    GPhoto2 también proporciona útiles funciones de depuración para los desarrolladores de controladores de cámaras.

    Comandos rápidos

    • gphoto2 --list-ports
    • gphoto2 --auto-detect
    • gphoto2 --summary
    • gphoto2 --list-files
    • gphoto2 --get-all-files
    • gphoto2 --set-config datetime=now - sets the camera to the current time

    Para la manipulación avanzada de archivos, utilice

    • gphoto2 --shell

    Ejemplo de uso con GVfs

    Autodetecte la cámara conectada y liste el puerto requerido: 

    $ gphoto2 --auto-detect
Model                          Port                                            
----------------------------------------------------------
Canon Digital IXUS 980 IS      usb:006,011

    Ahora abra su gestor de archivos favorito e introduzca la dirección del puerto encontrado con el detalle "gphoto2://[usb:006,011]" - la cámara se montará con GVfs y podrá ser gestionada con el gestor de archivos.

    Problemas de permisos

    Los usuarios con una sesión local tienen concedidos permisos para las cámaras mediante ACLs. Véase General troubleshooting (Español)#Permisos de sesión si no funciona.
