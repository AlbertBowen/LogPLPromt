    Questo documento vuole essere una guida per l'installazione di Arch Linux utilizzando un sistema live avviato da un supporto di installazione creato con un'immagine di installazione ufficiale.Il supporto di installazione fornisce funzionalità di accessibilità descritte nella pagina Install Arch Linux with accessibility options. Per metodi di installazione alternativi, vedere Category:Installation process.

    Prima di installare, è consigliato dare una lettura alle FAQ. Per le convenzioni usate leggere Help:Reading. In particolare, gli esempi di codice potrebbero contenere dei segnaposto (formattati in corsivo) che devono essere sostituiti manualmente.

    Questa guida è concisa ed è consigliato seguire le istruzioni per sezione nell'ordine in cui sono presentate. Per istruzioni più dettagliate sui vari programmi vedere le rispettive pagine ArchWiki e le man page dei programmi, entrambe linkate in questa guida. Per un aiuto interattivo sono anche disponibili i canali IRC e il forum internazionale.

    Arch Linux dovrebbe funzionare su qualsiasi computer compatibile x86_64 con un minimo di 512 MiB di RAM, ma l'avvio del sistema live per l'installazione richiede più memoria.[1] Un'installazione basilare dovrebbe occupare meno di 2 GiB di spazio su disco. Poiché il processo di installazione necessita di recuperare pacchetti da un repository remoto, questa guida assume la presenza di una connessione internet funzionante.

    Prima dell'installazione

    Recuperare un'immagine di installazione

    Il supporto di installazione e le sue firme GnuPG possono essere ottenute dalla pagina Download, a seconda di come si vuole procedere sono disponibili immagini ISO o Netboot.

    Verifica della firma

    E' raccomandata la verifica della firma dell'immagine prima dell'utilizzo, specialmente se è stata scaricata da un mirror HTTP, dove i download possono essere soggetti ad intercettazioni per fornire immagini malevoli.

    Su un sistema con GnuPG installato, eseguire il comando seguente per scaricare la firma PGP ISO (sotto Checksums nella pagina Download) nella directory col file ISO, e verificarla con: 

    $ gpg --keyserver-options auto-key-retrieve --verify archlinux-version-x86_64.iso.sig

    In alternativa, da un'installazione esistente di Arch Linux eseguire: 

    $ pacman-key -v archlinux-version-x86_64.iso.sig
    Nota:
    • La stessa firma potrebbe essere stata manipolata se è stata scaricata da un sito mirror, anziché dalla pagina archlinux.org come indicato sopra. In questo caso, assicurarsi che la chiave pubblica, che è utilizzata per decifrare la firma, sia firmata da un'altra chiave di fiducia. Il comando gpg riporterà l'impronta (fingerprint) della chiave pubblica.
    • Un altro metodo per verificare l'autenticità della firma è assicurarsi che l'impronta della chiave pubblica sia identica all'impronta della chiave dello Sviluppatore di Arch Linux che ha firmato il file ISO. Vedi Wikipedia:Public-key cryptography per avere più informazioni sul processo di autentificazione della chiave pubblica.

    Preparare un supporto di installazione

    L'ambiente live può essere avviato da una memoria USB, un disco ottico o da una rete con PXE. Seguite le relative pagine per preparare voi stessi il supporto all'installazione in base all'immagine da voi scelta.

    Avvio dell'ambiente live

    Nota: Le immagini di installazione di Arch Linux non supportano Secure Boot. È necessario disabilitare Secure Boot per avviare il supporto di installazione. Se lo desiderate, potete attivare Secure Boot dopo aver completato l'installazione.
    1. Indicare come dispositivo di avvio l'unità contenente il supporto di installazione di Arch Linux. Ciò è tipicamente eseguito premendo un tasto specifico durante la fase POST, come indicato sulla schermata di avvio. Fare riferimento al manuale della propria scheda madre per i dettagli.
    2. Quando appare il menu boot loader del supporto di installazione, selezionare Arch Linux install medium e premere Invio per accedere all'ambiente di installazione.
      Suggerimento: L'immagine di installazione utilizza GRUB per l'avvio UEFI e syslinux per l'avvio BIOS. Vedi README.bootparams per una lista di parametri di avvio.
    3. Eseguirai l'accesso alla prima console virtuale come l'utente root, e ti si presenterà una shell dei comandi Zsh.

    Per cambiare ad una console differente-ad esempio, per leggere questa guida con Lynx in parallelo con l'installazione-utilizzare la scorciatoia tastiera Alt+freccia. Per modificare i files di configurazione sono disponibili mcedit(1), nano e vim. Vedere pkglist.x86_64.txt per una lista dei contenuti inclusi nel supporto di installazione.

    Impostare la disposizione della tastiera nella console

    La mappatura della tastiera predefinita è quella US. Le disposizioni disponibili possono essere elencate con: 

    # ls /usr/share/kbd/keymaps/**/*.map.gz

    Per impostare la disposizione della tastiera, aggiungere il nome del file corrispondente al comando loadkeys(1), omettendo percorso ed estensione del file. Ad esempio, per impostare la mappatura della tastiera Tedesca: 

    # loadkeys de-latin1

    I fonts per la console si trovano in /usr/share/kbd/consolefonts/ e possono essere impostati analogamente con setfont(8).

    Verificare la modalità di boot

    Per verificare con quale modalità di avvio , elencare la directory efivars: 

    # ls /sys/firmware/efi/efivars

    Se il comando restituisce un elenco senza errori allora il sistema si è avviato in modalità UEFI. Se la directory non esiste, il sistema potrebbe essere avviato in modalità BIOS o CSM. Fare riferimento al manuale della scheda madre per i dettagli.

    Connettersi ad Internet

    Per impostare una connessione di rete nell'ambiente live, eseguire i passaggi seguenti:

    • Assicurarsi che l'interfaccia di rete sia elencata ed abilitata, per esempio con ip-link(8): 
      # ip link
    • Per reti wireless e WWAN, assicurarsi che la scheda non sia bloccata con rfkill.
    • Connettersi alla rete:
      • Ethernet-inserire il cavetto.
      • WI-FI-autenticarsi alla propria rete wireless tramite iwctl.
      • Modem di rete mobile a banda larga-connettersi alla rete mobile utilizzando l'utilità mmcli.
    • Configurate la vostra connessione di rete:
      • DHCP: assegnazione di indirizzo di ip dinamico e del server DNS (fornito da systemd-networkd e systemd-resolved) dovrebbe funzionare tranquillamente per le interfacce di rete Ethernet, WLAN e WWAN.
      • Indirizzo IP statico: vedere la pagina Configurazione di rete#Indirizzo_IP_statico.
    • La conessione può essere verificata usando ping: 
      # ping archlinux.org
    Nota: Nell'immagine di installazione systemd-networkd, systemd-resolved, iwd e ModemManager sono preconfigurati e abilitati per impostazione predefinita. Non sarà così per il sistema installato.

    Aggiornare l'orologio di sistema

    systemd-timesyncd è attivo per pre-impostazione nell'ambiente live e l'orario sarà sincronizzato automaticamente una volta che viene stabilita una connessione ad Internet.

    Utilizzare il comando timedatectl(1) per assicurarsi che l'orologio sia corretto: 

    # timedatectl status

    Partizionare il disco

    Quando riconosciuti dal sistema live, i dischi sono assegnati a dispositivi a blocchi come /dev/sda, /dev/nvme0n1 o /dev/mmcblk0. Per identificare questi dispositivi, utilizzare lsblk o fdisk. 

    # fdisk -l

    I risultati che terminano con rom, loop o airoot potrebbero essere ignorati.

    Le seguenti partizioni sono richieste su un dispositivo scelto per l'installazione:

    • Una partizione per la cartella|radice] (o root) /.
    • Per l'avvio in modalità UEFI, una partizione EFI di sistema.

    Se si desidera creare un dispositivo a blocchi impilabile per LVM, per la cifratura del sistema o per il RAID, è necessario farlo adesso.

    Utilizzare fdisk o parted per modificare le tabelle delle partizioni, per esempio: 

    # fdisk /dev/disco_che_si_vuole_partizionare
    Nota:
    • Se 'l'unità non viene mostrata, assicurarsi che il controllo dei dischi non sia in modalità RAID
    • Se nel disco da cui vuoi avviare il sistema è già presente una partizione EFI di sistema, usa quella esistente, non crearne un'altra.
    • Lo spazio per lo swap può essere impostato su un file di swap per quei file system che lo supportano.

    Disposizioni di esempio

    UEFI con GPT
    Punto di montaggio Partizione Tipo di partizione Dimensione suggerita
    /mnt/boot1 /dev/partizione_di_sistema_efi Partizione di sistema EFI Almeno 300 MiB. Se sono installati kernel multipli, almeno 1 GiB.
    [SWAP] /dev/partizione_per_swap Linux swap Maggiore di 512 MiB
    /mnt /dev/partizione_per_root Linux x86-64 root (/) Resto del dispositivo
    1. Altri punti di montaggio, come /mnt/efi, dovrebbero essere considerati solo se il boot loader che si intende utilizzare è in grado di caricare il kernel e le immagini initramfs dal volume di root. Vedi l'avviso in Arch boot process#Boot loader.
    BIOS con MBR
    Punto di montaggio Partizione Tipo di partizione Dimensione suggerita
    [SWAP] /dev/partizione_per_swap Linux swap Maggiore di 512 MiB
    /mnt /dev/partizione_per_root Linux Resto del dispositivo

    Vedere anche Creare partizioni#Disposizioni_di_esempio.

    Formattare le partizioni

    Ciascuna delle partizioni appena create deve essere formattata con un file system appropriato. Vedasi File systems#Creare_un_filesystem per i dettagli.

    Per esempio, se la partizione della radice è /dev/partizione_per_root, per creare un file system ext4, esegui: 

    # mkfs.ext4 /dev/partizione_per_root

    Se hai creato una partizione di swap, inizializzala con mkswap(8): 

    # mkswap /dev/partizione_per_swap
    Nota: Per i dispositivi a blocchi impilabili, sostituisci /dev/partitione_per_* con il percorso del dispositivo a blocchi appropriato.

    Se hai creato una partizione di sistema EFI, formattala in FAT32 usando mkfs.fat(8).

    Attenzione: Formattare la partizione di sistema EFI solo se è stata creata durante la fase di partizionamento. Se in precedenza era già presente una partizione di sistema EFI sul disco, la riformattazione può distruggere i boot loader di altri sistemi operativi installati.
    # mkfs.fat -F 32 /dev/partitione_di_sistema_efi

    Montare il file system

    Montare il volume di root in /mnt. Ad esempio, se il volume di root è /dev/partizione_per_root : 

    # mount /dev/partizione_per_root /mnt

    Creare eventuali punti di montaggio rimanenti (come /mnt/boot) e montarne i volumi nell'ordine gerarchico corrispondente.

    Suggerimento: Eseguire mount(8) con l'opzione --mkdir per creare il punto di montaggio specificato. Altrimenti, crearlo prima con l'ausilio di mkdir(1).

    Per i sistemi UEFI, montare la partizione di sistema EFI: 

    # mount --mkdir /dev/partitione_di_sistema_efi /mnt/boot

    Se hai creato un volume di swap, abilitatelo con swapon(8): 

    # swapon /dev/partizione_per_swap

    genfstab(8) successivamente rileverà file systems montati e spazio di swap.

    Installazione

    Selezionare il mirror corretto

    I pacchetti da installare verranno scaricati da server mirror che sono definiti in /etc/pacman.d/mirrorlist. Sul sistema live, dopo aver stabilito la connessione di rete, reflector aggiornerà tutta la lista dei server mirror scegliendo gli ultimi 20 recentemente aggiornati e sincronizzati , ordinandoli in abase al loro download rate [2].

    Il mirror che si troverà più in alto nella lista sarà quello a priorità maggiore quando si andrà a scaricare un pacchetto. Potresti voler modificare questo file pe verificare se ti soddisfa. Nel caso,editate il file a vostro piacimento e spostare il mirror più vicino geograficamente in cima alla lista, oppure secondo altri criteri.

    Questo file, verrà poi copiato sul nuovo sistema installato da pacstrap, pertanto è importante modificarlo correttamente.

    Installare i programmi essenziali

    Nota: Nessun software o configurazione (ad eccezione di /etc/pacman.d/mirrorlist) viene mantenuta dall'ambiente live al sistema installato.

    Usare lo script pacstrap(8) per installare il contenuto base, Linux kernel e il firmware per gli hardware più comuni: 

    # pacstrap -K /mnt base linux linux-firmware
    Suggerimento:
    • Puoi sostituire il pacchetto linux con un contenuto kernel a tua scelta, oppure potresti ometterli del tutto quando installi in un container.
    • Potresti omettere l'installazione del pacchetto dei firmware se si sta installando in una macchina virtuale o in un contenitore.

    Il contenuto base non include tutti gli strumenti presenti sull'installazione live, quindi potrebbe rendersi necessaria l'installazione di ulteriori contenuti per un sistema base completamente funzionante. Per installare altri contenuti o gruppi di contenuti, aggiungere i nomi (separati da spazio) al comando pacstrap visto in precedenza o utilizzare pacman dopo aver eseguito il chroot dentro il nuovo sistema.

    • utilità in spazio utente per la gestione dei file systems che saranno utilizzati sul sistema,
    • utilità per accedere a partizioni RAID o LVM,
    • firmware specifico per altri dispositivi non incluso in linux-firmware (es. sof-firmware per schede audio),
    • software necessari per la rete (es. un gestore di rete o un client DHCP autonomo), authentication software per Wi-Fi, ModemManager per connessioni mobili a banda larga),
    • un editor di testo,
    • programmi per accedere alla documentazione nelle pagine man e info: man-db, man-pages e texinfo.

    Per confrontare, i programmi disponibili nel sistema live si trovano su pkglist.x86_64.txt.

    Configurare il sistema

    Fstab

    Generare un file fstab (usare -U o -L per definire se si vuole riferire alla partizioni con UUID o labels rispettivamente): 

    # genfstab -U /mnt >> /mnt/etc/fstab

    Controllare il risultato nel file /mnt/etc/fstab ed editare in caso di errore.

    Chroot

    Cambiare la radice nel nuovo sistema: 

    # arch-chroot /mnt

    Fuso orario

    Impostare il proprio fuso orario: 

    # ln -sf /usr/share/zoneinfo/Regione/Città /etc/localtime

    Eseguire hwclock(8) per generare /etc/adjtime: 

    # hwclock --systohc

    Questo comando assume che il proprio orologio hardware sia impostato su UTC. Per ulteriori dettagli, vedere System time#Time standard.

    Localizzazione

    Modificare /etc/locale.gen e decommentare it_IT.UTF-8 UTF-8 e altre localizzazioni necessarie. Generare le localizzazioni eseguendo: 

    # locale-gen

    Creare il file locale.conf(5), e impostare la variable LANG di conseguenza: 

    /etc/locale.conf
    LANG=it_IT.UTF-8

    Se si è settato un layout della tastiera, rendere persistente questa modifica in vconsole.conf(5): 

    /etc/vconsole.conf
    KEYMAP=it

    Configurazione della rete

    Creare il file hostname(5): 

    /etc/hostname
    mio_hostname

    Completa la configurazione di rete per il nuovo ambiente installato, che include l'installazione del software preferito per la gestione della rete.

    Initramfs

    La creazione di un nuovo initramfs normalmente non è necessaria, in quanto mkinitcpio è stato eseguito durante l'installazione del pacchetto kernel con pacstrap.

    Per LVM, cifratura del sistema o RAID, modificare mkinitcpio.conf(5) e ricreate l'immagine initramfs: 

    # mkinitcpio -P

    Password di root

    Impostare una password per l'utente root: 

    # passwd

    Boot loader

    Scegli e installa un boot loader compatibile con Linux. Se hai una CPU Intel o AMD, abilitane l'aggiornamento del microcodice.

    Reboot

    Uscire dall'ambiente chroot con exit o premendo Ctrl+D.

    Opzionalmente smontare a mano le partizioni con umount -R /mnt: questo potrebbe portare a qualche warning di partizione "busy" (occupata, quindi impossibile da smontare), trovarne la causa con fuser(1).

    Riavviare la macchina con un reboot. Tutte le partizioni vengono smontate automaticamente da systemd. Ricordare di rimuovere il supporto di installazione e di loggarsi sulla nuova Arch Linux tramite l'utente root appena creato.
